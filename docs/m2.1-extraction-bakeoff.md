The replay fixture is intentionally deterministic and exists so default CI can test
schema validation, matching logic, cost calculation, and reporting without calling
DeepSeek.

The fact-level gold file is an exhaustive draft label set. The benchmark now reports
a scorecard instead of legacy tuple precision/recall:

- `grounded_precision`: claims grounded in the same chunk text divided by total claims.
- `coverage_recall`: chunk-scoped matched gold facts divided by all gold facts.
- `primary_coverage_recall`: the same recall restricted to primary facts.
- `numeric_grounding_rate`, `direction_accuracy`, `magnitude_accuracy`, and
  `subject_accuracy`: separate diagnostic axes.
- `redundancy_rate`: grounded claims that did not cover a distinct fact.

## Live Benchmark Status

Live benchmark outputs are local artifacts and are ignored by git under
`data/benchmark/results/*.json`. Commit only stable regression fixtures under
`tests/fixtures/`.

The scorecard regression fixture is
`tests/fixtures/haiku-scorecard-regression.json`. Its top-level metrics were produced
before the scorecard evaluator. Reanalyzing the saved responses with the current
evaluator gives `schema_validation_rate=0.90`, `grounded_precision=0.67`,
`coverage_recall=0.49`, `primary_coverage_recall=0.54`, and
`numeric_grounding_rate=0.93`. The run still fails the gate because schema validation,
grounded precision, and coverage recall miss their targets.

One fixture chunk failed schema validation: `chunk-07-investment-gains-tax` returned
`macro_variable="RATE"` on a non-`MACRO_EXPOSURE` claim. Before the evaluator fix, the
global multiset match let other chunks cover that miss. The current scorecard matches
claims to facts only within the same chunk, so that failed chunk remains uncovered.

A later DeepSeek live run completed locally with `schema_validation_rate=1.00`,
`grounded_precision=0.739`, `coverage_recall=0.633`, and
`cost_per_chunk_usd=0.000408`. The raw result JSON is intentionally not committed.

Run the DeepSeek live benchmark manually:

```bash
set -a; source .env; set +a
python -m dcf_engine.extraction.benchmark
```

Re-run the Claude Haiku live benchmark after prompt/schema changes:

```bash
set -a; source .env; set +a
python -m dcf_engine.extraction.benchmark --provider anthropic
```

Live commands write `data/benchmark/results/{model}__{timestamp}.json`.

## Decision

Replay validation exercises the CI harness deterministically, but it no longer
represents a passing model gate because the fixture predates fact-level exhaustive
gold. The model viability decision remains pending until `data/benchmark/gold_facts.json`
is reviewed and frozen. Claude Haiku 4.5 does not pass the current scorecard gate with
prompt `m2.1-v1`.
