The replay fixture is intentionally deterministic and exists so default CI can test
schema validation, matching logic, cost calculation, and reporting without calling
DeepSeek.

The draft gold file is a seed-label set, not an exhaustive list of every valid claim.
For that label status the benchmark now reports coverage semantics: extra model
claims are treated as unevaluated instead of false positives, and precision uses the
same seed-gold denominator as recall. Matching is scoped by `chunk_ref`,
`claim_subject`, `direction`, and `magnitude_qualifier` so one chunk can no longer hide
another chunk's miss.

any benchmark result could be recorded.

Claude Haiku live execution completed and wrote
`data/benchmark/results/claude-haiku-4-5-20251001__20260607T081235Z.json`. That file's
top-level metrics were produced before the chunk-aware evaluator fix. Reanalyzing the
saved responses with the current evaluator gives `schema_validation_rate=0.90`,
`precision=0.50` under partial-gold coverage semantics, and `recall=0.50`. The run
still fails the frozen gate because schema validation and recall both miss the target.

One chunk failed schema validation: `chunk-07-investment-gains-tax` returned
`macro_variable="RATE"` on a non-`MACRO_EXPOSURE` claim. Before the evaluator fix, the
global multiset match let other chunks cover that miss and inflated recall from the
chunk-aware `0.50` result to `0.80`.
